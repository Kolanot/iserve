MetaCarta_Location_Finder

The Location Finder offers output in 3 formats:

  • JSON -- The most fully featured format, the JSON format includes all the available information about the locations.
  • KML -- A more lightweight format, but suitable for use in applications which use KML.
  • PNG -- An image output format, which will return an image with the locations returned displayed on the image.

To form a request URL:

  • Start with http://ondemand.metacarta.com/webservices/LocationFinder/
  • Append the format, one of JSON, KML, or PNG: http://ondemand.metacarta.com/webservices/LocationFinder/JSON
  • Choose an authentication method: one of session or basic: http://ondemand.metacarta.com/webservices/LocationFinder/JSON/basic
  • Append the appropriate parameters to the request, as described below.

Examples of Usage

  • Javascript API Client: LocationFinder Lookup
  • The Python Client comes with a built in LocationFinder example.

Request Parameters

version:
Required
default value: input other than the allowed value raises an error message
allowed values: 1.0.0
usage: The requestor must specify the API version they wish to use. The API requires a valid version parameter.
query:
Required
default value: ''
allowed values: string
usage: The query string is the input for which a LocationFinder search is initiated.
geometry:
Optional
default value: None
allowed values: true, false
usage: If this is set to true, the LocationFinder will return geometries for features when available.
width:
Optional
default value: 512
allowed values: positive integer
usage: Determines the width of the image returned by the PNG LocationFinder output.
height:
Optional
default value: 383
allowed values: positive integer
usage: Determines the height of the image returned by the PNG LocationFinder output.
