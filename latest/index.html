About iServe

iServe is what we refer to as service warehouse which unifies service publication, analysis, and discovery through the use of lightweight semantics as well as advanced discovery and analytic capabilities. iServe provides the typical features of service registries and additional functionality that exploits service descriptions, service annotations and further data gathered and derived from the analysis of these descriptions, data crawled from the Web, periodic monitoring and user activities. iServe builds upon lessons learnt from research and development on the Web and on service discovery algorithms to provide a generic semantic service registry able to support advanced discovery over both Web APIs and WSDL services described using heterogeneous formalisms.

Project Modules

This project has declared the following modules:

Name Description
iServe Commons I/O iServe Commons I/O provides basic support for reading and writing service descriptions.
iServe Commons Vocabulary iServe Commons Vocabulary provides a Java object model for the manipulation of service descriptions in terms of the vocabulary used within iServe.
iServe SAWSDL Importer iServe SAWSDL Importer provides support for transforming SAWSDL service descriptions into the model used by iServe, namely Minimal Service Model.
iServe hRESTS Importer iServe hRESTS Importer provides support for transforming hRESTS service descriptions into the model used by iServe, namely Minimal Service Model.
iServe OWL-S Importer iServe OWL-S Importer provides support for transforming OWL-S service descriptions into the model used by iServe, namely Minimal Service Model.
iServe WSC 2008 Importer iServe WSC 2008 Importer provides support for transforming service descriptions in terms of the proprietary format of the WSC 2008 contest into the model used by iServe, namely Minimal Service Model.
iServe Storage Access Layer (SAL) Core iServe Storage Access Layer (SAL) Core provides the core of iServe including all data management interfaces and their corresponding implementations.
iServe SAL REST iServe SAL REST provides a RESTful interface to the registry exposing the Linked Data hosted by the registry either for humnans (HTML) or for machines (RDF, JSON) as well as it enables applications to create, read, update or delete data from the server.
iServe Discovery API iServe Discovery API provide the core infrastructure and APIs for the development and management of discovery plugins.
iServe Disco Discovery Plug-in iServe Disco provides a default implementation of discovery plugins exploiting logic-based discovery.
iServe Discovery Engine iServe Discovery Engine provides the basic machinery for exposing discovery plugins via a RESTful interface. It additionally provides support for combining diverse plugin results (Union, Intersection and Subtraction).
iServe Shared Test Resources iServe Shared Test Resources is a convenience module capturing some of the resources used for tests within the project, as well as providing direct scripts for automatically downloading other resources from the Web.
